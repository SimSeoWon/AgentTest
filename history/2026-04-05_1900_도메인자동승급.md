# 도메인 자동 승급 기능 추가

## 작업 개요
검색 로그의 동시 출현 패턴을 분석하여, 자주 함께 검색되는 컨텍스트 문서들을
종합적인 "도메인 문서"로 자동 승급시키는 기능을 구현했다.

## 배경
- 개별 컨텍스트 MD는 파일별 간단한 요약 + 파일 참조 (데이터)
- 자주 함께 조회되는 문서들은 하나의 기능/시스템을 구성 → 종합 문서 필요 (정보)
- 도메인 문서를 통해 추후 하네스/스킬에서 프로젝트 맥락으로 능동 활용 가능

## 설계

### 데이터 흐름
```
에이전트 combined_search 호출
  → context_search.exe가 결과 문서 ID를 search_log.jsonl에 기록
  → watch.py 폴링 루프에서 주기적으로 분석 (DOMAIN_CHECK_INTERVAL=10)
  → 동시 출현 ≥ 5회인 문서 클러스터 식별 (Union-Find)
  → LLM으로 도메인 문서 생성 → context/_domains/<이름>.md
  → 벡터 인덱스에 자동 포함 (rglob("*.md"))
```

### 도메인 문서 스키마
```markdown
---
tags: [미션, 퀘스트, 태스크]
category: 도메인/미션시스템
type: domain
source_documents:
  - Source/Mission/TaskSystem.md
  - Source/Mission/MissionManager.md
---

## 시스템 개요
## 클래스 간 관계
## 데이터 흐름
## 설계 패턴 및 확장 포인트
```

## 변경 파일

### mcp/context_search/server.py
- `_log_search()` 함수 추가 — 검색 결과 문서 ID를 `.claude/search_log.jsonl`에 기록
- `combined_search` (MCP tool) 반환 직전에 로깅 호출
- `api_combined_search` (HTTP 핸들러) 반환 직전에 로깅 호출

### watcher/watch.py
- 상수: `DOMAIN_DIR_NAME`, `COOCCURRENCE_THRESHOLD`, `DOMAIN_MIN_DOCS`, `DOMAIN_CHECK_INTERVAL`
- `_analyze_search_patterns()` — 검색 로그 분석, 동시 출현 매트릭스, Union-Find 클러스터링
- `_get_existing_domains()` — 기존 도메인 문서의 소스 문서 목록 파싱
- `promote_domains()` — 클러스터 → LLM 종합 → 도메인 문서 생성 + 벡터 인덱싱
- 폴링 루프에 `poll_count` 카운터 추가, 매 10회마다 `promote_domains()` 호출

### CLAUDE.md
- 도메인 자동 승급 설명 추가

## 주요 설계 결정
| 결정 | 이유 |
|------|------|
| combined_search에만 로깅 | 에이전트가 항상 우선 사용하는 함수, 중복 방지 |
| 결과 2개 미만은 로깅 안 함 | 동시 출현 분석에 무의미 |
| Union-Find 클러스터링 | 외부 라이브러리 불필요, O(α(n)) 성능 |
| 80% 겹침으로 중복 판단 | 소스 문서가 약간 달라도 같은 도메인으로 인정 |
| _domains/ 하위 디렉토리 | rglob("*.md")로 자동 인덱싱, context_search 수정 불필요 |
| 폴링 10회마다 체크 | 검색 데이터가 충분히 쌓인 후 분석 |
