## 확장성 평가

| 팀 규모 | 권장 모드 | 주요 병목 | 대응 필요 |
|---------|-----------|-----------|-----------|
| **6~8명** | 로컬 | 저사양 PC 벡터 인덱싱 지연 | 현재 구조로 충분 |
| **20명** | 서버 | FastAPI 단일 워커, rebuild 중 검색 마비 | 다중 워커 필요 |
| **50명** | 서버 | HTTP 504 에러, 브랜치별 서버 필요 | 아키텍처 재설계 필요 |

## 즉시 수정 권장 (Top 5)

1. **config.json 로드 시 타입/범위 검증** — 크래시 방지
2. **watch_state 파일 encoding='utf-8' 명시** — 한글 경로 크래시 방지
3. **포트 점유 감지 + 기존 프로세스 정리** — 재시작 보장
4. **Git pull 실패 시 상태 판별 + 사용자 알림** — 무한 대기 방지
5. **초기 캐시 warming** — 클라이언트 서버 미응답 시 최소한의 검색 보장
