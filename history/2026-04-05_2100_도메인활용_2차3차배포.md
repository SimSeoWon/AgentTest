# 도메인 활용 2차/3차 배포 (F3~F10)

## 작업 개요
도메인 문서 활용 전략의 나머지 기능들을 모두 구현했다.

## 2차 배포

### F3. 04_코드작성 에이전트 도메인 인식
- `agent_templates.py`의 ROLE/PROMPT/SETTINGS_TEMPLATES 수정
- context-search MCP 추가, "구현 전 도메인 검색" 지침 추가
- 기존 배포 프로젝트는 에이전트 폴더 삭제 후 재실행 필요

### F4. hooks 기반 도메인 자동 로딩
- `DOMAIN_HINT_SCRIPT`: PostToolUse + Edit|Write hook 스크립트
- `_deploy_hook_scripts()`: .claude/hooks/domain_hint.py 자동 배포
- `_update_project_settings()`에 hooks 키 머지 로직 추가
- 파일 편집 시 context_search.exe로 관련 도메인 검색 → stderr로 힌트 출력

### F10. 도메인 문서 코멘트 보존
- `promote_domains()`에서 도메인 재생성 시 기존 ## 코멘트 섹션 보존
- 기존 `_extract_comments_section()` 패턴 재활용

## 3차 배포

### F5. 영향 분석 MCP 도구
- `context_search/server.py`에 `impact_analysis(system_name)` MCP 도구 추가
- 도메인 매칭 → source_documents 1차 영향 → related_classes 2차 영향
- 확장 포인트 함께 반환

### F6. 프로젝트 건강도 리포트
- `generate_health_report()`: 도메인별 기술 부채/코멘트/미결 고민/리뷰 빈도 집계
- LLM 호출 없이 파일 파싱만으로 동작
- `reviews/_health_report.md`로 저장
- 매 50회 폴링마다 자동 갱신 (HEALTH_CHECK_INTERVAL)

### F9. 온보딩 아키텍처 맵
- `_generate_architecture_overview()`: 도메인 3개 이상 시 LLM으로 종합
- `_domains/_overview.md`에 저장 (프로젝트 구조 요약 + 시스템 간 관계 + 핵심 진입점)
- promote_domains에서 새 도메인 생성 시 함께 갱신
- 기존 코멘트 보존

## 변경 파일
- `watcher/watch.py`: F4, F6, F9, F10 구현
- `watcher/agent_templates.py`: F3 (04_코드작성 에이전트 수정)
- `mcp/context_search/server.py`: F5 (impact_analysis 도구 추가)
- `CLAUDE.md`: 배포 구조, MCP 도구 목록 갱신
