# 2026-03-29 13:00 — 컨텍스트 생성을 디렉토리(모듈) 단위로 그룹핑

## 작업 개요
컨텍스트 MD 생성 단위를 클래스(파일) 단위에서 디렉토리(모듈) 단위로 변경.
같은 폴더의 .h/.cpp 파일들을 하나의 프롬프트로 합쳐서 분석, 하나의 MD를 생성한다.

## 변경 전후

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| 그룹핑 단위 | .h/.cpp 쌍 (클래스) | 부모 디렉토리 (모듈) |
| 22개 파일 예시 | Claude 12회 호출, 12개 MD | Claude 5회 호출, 5개 MD |
| 프롬프트 크기 | 파일당 4000자 | 그룹당 8000자 |
| 출력 경로 | `context/.../MissionMultiTaskBase.md` | `context/.../TaskSystem.md` |
| 벡터 인덱싱 | 중복 ID로 에러 발생 | 디렉토리 MD + 하위호환 체크 |

## 기대 효과
- Claude 호출 횟수 ~60% 감소 (비용·시간 절약)
- 같은 모듈의 클래스 관계를 함께 분석 → 컨텍스트 품질 향상
- .h/.cpp 중복 MD 생성 문제 근본 해결

## 생성/수정 파일 목록

| 파일 | 변경 내용 |
|------|----------|
| `watcher/watch.py` | `_group_by_directory()` 도입, `update_context()` 디렉토리 단위로 전환, `GROUP_CONTENT_LIMIT=8000`, 벡터 인덱싱 경로 대응 |
| `CLAUDE.md` | 감시 루프 설명 갱신 |

## 주요 설계 결정

| 결정 | 이유 |
|------|------|
| 부모 디렉토리를 그룹 키로 사용 | UE5 프로젝트에서 디렉토리가 모듈 경계 역할 |
| 8000자 제한 | 파일 개별 4000자 대비 그룹 분석에 충분, 비용도 개별 합산보다 적음 |
| .h를 먼저 정렬 | 인터페이스(선언) → 구현 순서로 LLM이 분석하면 정확도 상승 |
| 벡터 인덱싱에 하위호환 체크 유지 | 기존 클래스 단위 MD가 남아있을 수 있으므로 |
