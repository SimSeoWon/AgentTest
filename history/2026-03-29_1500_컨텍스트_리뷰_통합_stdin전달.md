# 2026-03-29 15:00 — 컨텍스트+리뷰 통합 처리 및 stdin 프롬프트 전달

## 작업 개요
컨텍스트 MD 생성과 코드 리뷰를 별도 LLM 호출에서 1회 통합 호출로 변경.
프롬프트를 명령줄 인수 대신 stdin으로 전달하여 Windows 32KB 제한 해결.

## 변경 전후 (22개 파일 기준)

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| LLM 호출 횟수 | 12회 (컨텍스트 6 + 리뷰 6) | **6회** (통합) |
| 소요 시간 | ~6분 | **~4분** |
| 처리 라운드 | 2라운드 (컨텍스트→리뷰) | **1라운드** |
| 프롬프트 전달 | 명령줄 인수 (32KB 제한) | **stdin** (무제한) |
| 메인 루프 | update_context → update_vector → run_code_review | **process_commit** 단일 호출 |

## 생성/수정 파일 목록

| `CLAUDE.md` | 감시 루프, 코드 리뷰 흐름, CLI 호출 방식 갱신 |

## 주요 설계 결정

| 결정 | 이유 |
|------|------|
| LLM 응답을 구분자(`=== CONTEXT_MD ===`/`=== CODE_REVIEW ===`)로 분리 | 한 번의 호출로 두 결과를 구조적으로 파싱 |
| stdin으로 프롬프트 전달 | Windows 명령줄 32KB 제한으로 통합 리포트 생성 시 WinError 206 발생 |
| `process_commit()` 단일 진입점 | 메인 루프 단순화, 컨텍스트+벡터+리뷰를 하나의 함수로 관리 |
| auto_review=false 시 기존 01_소스분석 프롬프트 사용 | 리뷰 불필요 시 컨텍스트만 생성하는 경량 모드 유지 |
