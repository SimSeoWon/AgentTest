# 도메인 활용 1차 배포 (F1 + F2 + F7 + F8)

## 작업 개요
도메인 문서를 능동적으로 활용하는 4가지 기능을 구현했다.
watch.py만 수정하여 agent_templates.py 변경 없이 즉시 적용 가능.

## 구현 항목

### F1. CLAUDE.md 도메인 맵 동적 주입
- `_build_domain_map_section()`: context/_domains/*.md를 스캔하여 도메인 목록 테이블 생성
- `_merge_claude_md_section()`에 `dynamic_section` 파라미터 추가
- `_update_project_claude_md()`에서 도메인 맵을 AgentWatch 마커 구역에 포함
- 폴링마다 자동 갱신 → 도메인이 추가/삭제되면 CLAUDE.md에 즉시 반영

### F2. 코드 리뷰 시 도메인 컨텍스트 자동 주입
- `_find_matching_domain()`: file_paths가 속한 도메인 문서의 핵심 섹션 추출 (1500자 이내)
- `_process_directory_group()`에서 `domain_notice`로 프롬프트에 주입
- 추출 대상: "시스템 개요" + "클래스 간 관계" + "설계 패턴"

### F7. stale 도메인 자동 정리
- `_cleanup_stale_domains()`: 소스 문서가 50% 이상 사라진 도메인을 자동 삭제
- `promote_domains()` 시작부에서 호출

### F8. 기획 요청 가이드 규칙
- F1의 도메인 맵 섹션 하단에 "기능 추가/변경 요청 시 워크플로우" 규칙 포함
- 1) combined_search로 도메인 검색 → 2) 확장 포인트 확인 → 3) 소스 코드 읽기

## 변경 파일
- `watcher/watch.py`: 4개 함수 추가/수정
- `CLAUDE.md`: 도메인 활용 설명 추가

## 이해관계자별 효과
| 이해관계자 | 해당 Feature | 효과 |
|-----------|-------------|------|
| 엔진팀 | F7 | stale 도메인 자동 정리로 품질 유지 |
| 프로그래머 | F2 | 코드 리뷰가 시스템 수준 영향을 파악 |
| 기획자 | F8 | 기능 요청 시 도메인 기반 자동 안내 |
| 리더 | F1 | CLAUDE.md만 읽으면 프로젝트 구조 파악 |
