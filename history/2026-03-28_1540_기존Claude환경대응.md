# 2026-03-28 15:40 — 기존 Claude 환경 대응 (MCP 머지 + CLAUDE.md 갱신)

## 문제
기존에 Claude를 사용하던 프로젝트에 watch.exe를 배포하면:
- `.claude/settings.json`이 이미 존재 → MCP 서버가 등록되지 않음
- `CLAUDE.md`가 이미 존재 → context 구조/에이전트/MCP 안내가 추가되지 않음
- Claude가 `.claude/context/` 위치도 모르고, MCP 툴도 못 씀

## 해결 방법

### 신규 함수 (`watch.py`)
- `_update_project_settings(claude_dir)` — `.claude/settings.json`에 MCP 서버 머지
  - 이미 등록된 서버는 보존 (사용자 커스텀 유지)
  - 누락된 서버만 추가
  - 항상 실행 (최초/재실행 무관)
- `_update_project_claude_md(base_dir)` — `CLAUDE.md`에 AgentWatch 관리 구역 삽입/갱신
  - `<!-- AgentWatch:Start -->` ~ `<!-- AgentWatch:End -->` 마커로 우리 구역 식별
  - 마커 있으면: 해당 구역만 최신 내용으로 교체
  - 마커 없으면: 파일 끝에 추가
  - 파일 없으면: 새로 생성
  - 마커 외부 내용은 절대 수정하지 않음

### 신규 상수 (`agent_templates.py`)
- `MCP_SERVERS: dict[str, str]` — 서버명 → 실행 파일 경로 (4개)
- `AGENTWATCH_MD_MARKER_START` / `AGENTWATCH_MD_MARKER_END` — HTML 주석 마커
- `PROJECT_CLAUDE_MD_SECTION` — CLAUDE.md에 삽입할 AgentWatch 안내 섹션
  - context 파일 위치/형식
  - reviews 위치
  - 에이전트 목록 참고 경로
  - MCP 서버 4종 안내 표

### `init_project_dirs()` 변경
- 기존 `settings.json` 단순 생성 블록 제거
- `_update_project_settings(claude_dir)` 호출 추가
- `_update_project_claude_md(base_dir)` 호출 추가
- 두 함수 모두 항상 실행 (기존 파일 여부 무관)

## 수정 파일
- `watcher/watch.py`
- `watcher/agent_templates.py`
- `CLAUDE.md`

## 동작 시나리오
```
기존 .claude/ 있는 프로젝트에 watch.exe 배포
  → init_project_dirs() 실행
      ├── _update_project_settings()
      │     settings.json 로드 → mcpServers에 4개 머지 → 저장
      └── _update_project_claude_md()
            CLAUDE.md 로드 → 마커 있으면 교체, 없으면 추가 → 저장
```
