# 2026-03-28 17:00 — 벡터 RAG 검색 추가

## 작업 개요
컨텍스트 MD 파일을 ChromaDB 벡터 인덱스에 저장하고,
자연어 질의로 의미 기반 검색을 수행하는 RAG 시스템 구축.
기존 태그 기반 검색과 병행하여 하이브리드 검색 가능.

## 수정 파일
- `mcp/context_search/server.py` — 벡터 검색 3개 MCP 툴 + CLI 모드 추가
- `watcher/watch.py` — subprocess로 context_search.exe 호출하여 벡터 인덱싱
- `build.bat` — chromadb 의존성 + context_search.exe에만 히든 임포트 추가
- `watcher/agent_templates.py` — MCP 툴 설명 갱신, SKILL_INDEX RAG 안내 추가
- `CLAUDE.md` — 벡터 RAG 검색 섹션 추가

## 주요 설계

### exe 분리 전략
- **watch.exe** — chromadb 미포함, 가벼움 유지
- **context_search.exe** — chromadb 번들, 벡터 인덱싱 + 검색 전담
- watch.py → `context_search.exe --upsert` subprocess 호출로 위임

### context_search.exe 이중 모드
| 모드 | 호출 방법 | 사용처 |
|------|----------|--------|
| MCP 서버 | `context_search.exe` (인수 없음) | Claude 에이전트 |
| CLI | `--rebuild`, `--upsert`, `--status` | watch.py subprocess |

### 기술 스택
| 구성요소 | 선택 | 이유 |
|----------|------|------|
| 임베딩 | all-MiniLM-L6-v2 (ONNX) | ChromaDB 내장, PyTorch 불필요 (~2GB 절약) |
| 벡터 DB | ChromaDB PersistentClient | Python 네이티브, 파일 기반, PyInstaller 호환 |
| 유사도 | 코사인 거리 | 텍스트 유사도 표준 메트릭 |

### 인덱싱 전략
- **최초 실행**: `vector_db/` 미존재 시 `--rebuild`로 전체 인덱싱
- **커밋 감지**: 변경된 파일의 MD만 `--upsert`로 증분 갱신
- **수동 재구축**: `rebuild_index()` MCP 툴 또는 `--rebuild` CLI

### 신규 MCP 툴 (context_search)
| 툴 | 기능 |
|----|------|
| `vector_search(query, n_results, category_filter)` | 의미 기반 벡터 검색 |
| `rebuild_index(project_root)` | 전체 벡터 인덱스 재구축 |
| `index_status(project_root)` | 인덱스 상태 확인 (문서 수, 동기화 여부) |

### Graceful Degradation
- context_search.exe 없으면 watch.py가 벡터 인덱싱 건너뜀 (경고 로그)
- chromadb 미설치 시 MCP 툴이 에러 메시지 반환 (크래시 없음)
- 기존 태그 검색(`search_context`, `list_tags`)은 항상 정상 동작
