# 도메인 중복 생성 수정 + Windows 전용 명시

## 도메인 중복 생성 버그 수정

**문제**: 196개 문서 클러스터가 매 폴링마다 "도메인_5", "도메인_6"... 계속 생성됨.
**원인**: 중복 판정이 클러스터 기준 단방향(80%)만 체크. 기존 도메인(45개 소스)이 196개 클러스터의 80%(157개)에 미달하여 매번 "새 클러스터"로 판정.
**수정**: `domain.py` 양방향 체크 — 기존 도메인 소스의 80%가 클러스터에 포함되어도 중복으로 판정.
