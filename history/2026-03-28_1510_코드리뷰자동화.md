# 2026-03-28 15:10 — 코드 리뷰 자동화 기능 추가

## 작업 개요
git pull 후 변경된 파일에 대해 자동으로 코드 리뷰를 실행하고
`.claude/reviews/` 에 커밋별 리포트를 저장하는 기능 추가.

## 수정 파일
- `watcher/watch.py` 전체 재작성

## 주요 변경 내용

### 신규 함수
- `run_code_review()` — 코드 리뷰 파이프라인 진입점
- `_build_review_report()` — 07_코드매니저로 통합 리포트 생성
- `_call_claude()` — Claude CLI 호출 공통 헬퍼 (중복 코드 제거)

### 기존 함수 변경
- `init_project_dirs()` — `reviews/` 폴더 생성 추가, 반환값 3-tuple로 변경
- `load_or_init_config()` — `auto_review` 옵션 추가 (최초 설정 시 y/n 입력)
- `main()` — `auto_review` 설정에 따라 `run_code_review()` 조건부 호출

### 코드 리뷰 흐름
```
변경 파일 감지
  → update_context()   (01_소스분석 → context/ MD)
  → run_code_review()  (auto_review: true 시)
      ├── 파일별: 03_코드규약 + 05_코드검증
      └── 통합:   07_코드매니저
          → .claude/reviews/YYYY-MM-DD_HHMM_<hash>.md 저장
```

### config.json 변경
```json
{
  "branch": "main",
  "poll_interval": 60,
  "auto_review": true,   ← 신규
  "repo_dir": "..."
}
```
기존 config.json에 `auto_review` 없으면 기본값 `true` 적용.
