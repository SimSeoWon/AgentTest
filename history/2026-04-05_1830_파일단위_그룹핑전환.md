# 적응형 그룹핑 전환

## 작업 개요
컨텍스트 MD 생성의 그룹핑 전략을 적응형으로 변경했다.
디렉토리 단위를 기본으로 유지하되, 파일이 많아 크기를 초과하면 파일명(stem) 단위로 자동 분할한다.

## 배경
디렉토리 단위 그룹핑에서 파일이 많은 경우의 문제점:
- GROUP_CONTENT_LIMIT(8000자) 초과 시 뒤쪽 파일은 분석 누락
- 생성된 MD가 너무 커서 RAG 검색 결과가 에이전트 컨텍스트 윈도우를 과도하게 차지
- 500자 인위적 제한을 걸면 정보 손실 발생

## 변경 내용

### 핵심 변경: `_group_by_directory` → `_group_files` (적응형)
1단계: 디렉토리 단위로 그룹핑
2단계: 그룹 내 파일 합산 크기가 GROUP_CONTENT_LIMIT 초과 시 stem 단위로 분할

### 예시
```
소규모 디렉토리 (합산 < 8000자):
  Source/Utils/ → Utils.md (전체 합침)

대규모 디렉토리 (합산 > 8000자):
  TaskSystem.h + TaskSystem.cpp → Source/Mission/TaskSystem.md
  MissionManager.h + MissionManager.cpp → Source/Mission/MissionManager.md
```

### 변경 파일
- `watcher/watch.py`:
  - `_group_by_directory` → `_group_files` 교체 (적응형 하이브리드)
  - `process_commit`, `initial_context_build` 호출부 수정
  - `update_vector_index`: 디렉토리 MD + stem MD 모두 체크
- `watcher/agent_templates.py`: 이전에 추가한 500자/300자 제한 제거
- `CLAUDE.md`: 적응형 그룹핑 방식 설명 갱신

### 500자 제한 제거
적응형 분할로 각 MD가 자연스럽게 적절한 크기가 되므로 인위적 길이 제한 불필요.
