# 초기 컨텍스트 빌드 기능 추가

## 작업 개요
배포 파일 설치 후 최초 실행 시 기존 소스 파일 전체에 대한 컨텍스트 MD를 생성하고
벡터 인덱스를 구축하여 RAG를 초기화하는 기능을 추가했다.

## 배경
기존에는 `.watch_state`가 없으면 현재 HEAD 해시만 저장하고 다음 커밋부터 감시를 시작했다.
context/ 디렉토리가 비어있으므로 벡터 DB rebuild를 해도 인덱싱할 MD가 없어 빈 RAG 상태였다.

## 변경 파일
- `watcher/watch.py` — `_list_all_source_files()`, `initial_context_build()` 함수 추가, `main()` 초기화 흐름 수정
- `CLAUDE.md` — 초기 컨텍스트 빌드 설명 추가

## 주요 설계 결정

### 1. `git ls-files` 기반 파일 수집
- 파일시스템 탐색 대신 Git 추적 파일만 수집 (.gitignore 자동 존중)
- TARGET_EXTENSIONS 필터로 소스 파일만 추출

### 2. 누락분만 보충하는 증분 방식
- 모듈(디렉토리)별로 MD 존재 여부를 체크하여 없는 것만 생성
- 중단 후 재실행 시 처음부터 다시 하지 않고 남은 분만 처리
- 모든 MD가 이미 있으면 아무것도 하지 않고 즉시 반환

### 3. 기존 코드 최대 재활용
- `_group_by_directory`, `_process_directory_group` 등 기존 함수를 그대로 사용
- `auto_review=False`로 호출하여 컨텍스트 MD만 생성 (리뷰 불필요)
- `MAX_WORKERS=6` 병렬 처리 동일 적용

### 4. main() 흐름
- `init_project_dirs()` 이후, 폴링 루프 이전에 호출
- 기존 벡터 DB 체크 로직은 안전망으로 유지 (MD는 있지만 DB가 삭제된 경우 대비)
