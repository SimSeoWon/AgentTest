# 2026-03-29 14:00 — 예외 처리, 중복 실행 방지, 모델 선택, 프로세스 정리

## 작업 개요
병렬 워커 에러 핸들링, 중복 실행 방지 락, 작업 중 누적 커밋 처리,
Claude 모델 선택(sonnet/opus/haiku), Windows Job Object 프로세스 정리를 추가.

## 추가된 안전장치

### 1. 병렬 워커 에러 핸들링
- `update_context()`: 개별 그룹 실패 시 해당 그룹만 스킵, 나머지 계속 진행
- `run_code_review()`: 개별 파일 실패 시 해당 파일만 스킵, 나머지 계속 진행
- 실패 건수 로그 출력

### 2. 중복 실행 방지 (`_processing_lock`)
- `threading.Lock()`으로 동시 처리 방지
- 이전 작업 진행 중이면 `[대기] 이전 작업 진행 중` 로그 후 다음 폴링으로 넘김
- `finally` 블록에서 반드시 lock 해제

### 3. 작업 중 누적 커밋 처리
- `git pull` 후 `get_local_hash()`로 실제 최신 해시 재확인
- 작업 시작 시점과 pull 시점 사이에 추가 커밋이 있으면 diff 범위를 확장

### 4. Claude 모델 선택
- `config.json`에 `claude_model` 필드 추가 (기본값: `claude-sonnet-4-6`)
- 초기 설정 시 sonnet/opus/haiku 3택 메뉴 제공
- `_call_claude()`에서 `--model` 플래그로 전달

### 5. Windows Job Object 프로세스 정리
- `_setup_job_object()`: JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE로 자식/손자 프로세스 자동 정리
- `_kill_orphan_mcps()`: atexit + SIGBREAK 폴백으로 MCP 프로세스 taskkill
- CMD 창 닫기/작업관리자 강제종료 모두 대응
