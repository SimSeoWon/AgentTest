# 2026-03-29 12:30 — 자동 리뷰에 관련 컨텍스트 검색 추가

## 작업 개요
자동 코드 리뷰 시 변경 파일만 단독으로 분석하던 것을,
벡터+태그 통합 검색으로 관련 파일의 컨텍스트를 수집하여 리뷰 프롬프트에 주입하도록 개선.
파일 간 의존 관계를 이해한 리뷰가 가능해짐.

## 변경 전후

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| 코드 검증 프롬프트 | 해당 파일 컨텍스트 1000자만 참조 | + 관련 파일 컨텍스트 최대 3개 (파일당 400자) |
| context_search CLI | --rebuild, --upsert, --status | + `--search` 모드 추가 |
| 리뷰 데이터 흐름 | 파일 → 단독 분석 | 파일 → 관련 검색 → 교차 참조 분석 |

## 생성/수정 파일 목록

| 파일 | 변경 내용 |
|------|----------|
| `mcp/context_search/server.py` | `--search` CLI 모드 추가 (combined_search 호출) |
| `watcher/watch.py` | `search_related_contexts()`, `_build_related_context()` 추가, `run_code_review()`에 관련 컨텍스트 주입 |
| `CLAUDE.md` | 코드 리뷰 흐름 설명 갱신, CLI 모드 문서 갱신 |

## 주요 설계 결정

| 결정 | 이유 |
|------|------|
| CLI --search 모드로 호출 | watch.exe에 chromadb 미포함 유지 (경량 크기) |
| 관련 컨텍스트 3개, 파일당 400자 | 프롬프트 크기 제한 내에서 최대 맥락 확보 |
| 현재 파일 자신은 검색 결과에서 제외 | 중복 정보 방지 |
| 코드 검증(05)에만 관련 컨텍스트 주입 | 규약 검토(03)는 파일 단독으로 판단 가능 |
| 검색 쿼리 = 파일명 + 요약 200자 | 의미 기반 관련 파일 탐색 최적화 |
