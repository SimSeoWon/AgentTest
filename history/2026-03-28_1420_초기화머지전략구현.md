# 2026-03-28 14:20 — 초기화 머지 전략 구현

## 작업 개요
`watch.exe`가 이미 초기화된 프로젝트에 재배포될 때 기존 커스텀 설정을 보존하면서
새 에이전트/도메인만 추가하는 머지 방식으로 초기화 로직 개선.

---

## 수정 파일

- `watcher/watch.py` — `init_project_dirs`, `_init_agents` 로직 개선

---

## 변경 내용

### 문제
`_init_agents()`가 `if is_new:` 가드 안에 있어서, `.claude/`가 이미 존재하면
새 버전의 `watch.exe`에서 에이전트가 추가돼도 현장에 반영되지 않았음.

### 해결
- `_init_agents()` → `_merge_agents()`로 변경, 항상 실행
- 머지 전략:

| 대상 | 전략 |
|------|------|
| `.claude/settings.json` | 없을 때만 생성 (사용자 설정 보존) |
| `context/` 도메인 폴더 | 항상 `mkdir exist_ok=True` |
| 에이전트 폴더 | 항상 `mkdir exist_ok=True` |
| `role.md` / `prompt.md` / `settings.json` | 없을 때만 생성 (커스텀 보호) |
| `SKILL_INDEX.md` | 항상 덮어쓰기 (인덱스 최신 유지) |
| `config.json` | 없을 때만 생성 |

- 새로 추가된 에이전트가 있으면 로그에 출력:
  `.claude/ 머지 완료 — 새 에이전트 2개 추가: 08_로그분석, 09_크래시분석`
