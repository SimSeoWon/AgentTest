# 클라이언트 캐시/폴백 + 검색 개선

## 작업 개요

서버/클라이언트 모드에서의 검색 응답 개선, 클라이언트 로컬 캐시 및 폴백 처리 추가.

## 변경 내용

### 1. 검색 응답에 related_classes 추가
- ChromaDB metadata에 `related_classes`를 JSON 문자열로 저장
- 벡터/태그/combined 모든 검색 경로에서 `related_classes` 반환
- 에이전트가 관련 파일 경로를 즉시 파악 가능

### 2. combined_search 결과 수 제한
- MCP + HTTP API 모두 최종 결과를 `n_results`(기본 5개)로 제한
- 기존에는 태그 검색 결과가 무제한으로 합쳐져 불필요한 컨텍스트 윈도우 소비

### 3. 클라이언트 MD 캐시
- 원격 검색 성공 시 결과를 `.claude/context/<file>.md`에 저장
- 캐시 내용: frontmatter(tags, category, related_classes)만 저장 (본문 없음)
- 검색할 때마다 자동 갱신 — 별도 동기화 불필요

### 4. 서버 미응답 폴백
- 서버 응답 실패 시 `_fallback_tag_search()`로 캐싱된 MD에서 태그 기반 검색
- 쿼리를 토큰으로 분리 → 캐시 MD의 태그와 매칭 → 매칭 수 내림차순 정렬
- `combined_search`, `vector_search` 원격 호출 모두 적용

### 5. 웹 UI 추가
- `GET /` 에 HTML 페이지: 상태 카드, 검색, 문서 목록, 태그 클라우드
- 기존 API 엔드포인트를 JS에서 호출하는 방식 (백엔드 수정 최소)

### 6. 기타
- Gemini CLI 프롬프트 전달을 커맨드 인수 → stdin으로 변경 (Windows 명령줄 길이 제한 회피)
- 서버 모드 시 방화벽 인바운드 규칙 자동 확인/추가 기능
- 웹 UI 검색 결과에서 similarity 0%는 숨기고 source 표시

### 7. cache_context MCP 도구
- 에이전트가 소스 코드 직접 탐색 후 `cache_context(file_path, tags, category, related_classes, summary)` 호출
- 서버 응답 캐시와 동일한 `.claude/context/<file>.md` 경로에 저장
- 폴백 검색에서 서버 캐시 + 에이전트 캐시 모두 활용

### 8. 에이전트 행동 원칙 강화 (CLAUDE.md)
- "모든 판단은 소스 코드 기준" — RAG는 네비게이션만
- "탐색 결과는 cache_context로 저장" — 캐시 자동 축적
- "캐시도 검증 대상" — 소스로 재확인 필수

### 9. 버그 수정
- config.json 타입/범위 검증 (poll_interval int 변환, 최소 10초)
- .watch_state 파일 encoding='utf-8' 명시 (한글 경로 크래시 방지)
- git pull 실패 시 merge conflict/로컬 변경 구분 안내 메시지
- 웹 UI HTML을 web_ui.py로 분리 (server.py 1657→1371줄)

### 10. 날짜별 로그 시스템
- `.claude/logs/watch_YYYY-MM-DD.log` + `context_search_YYYY-MM-DD.log`
- 7일 보관, 자동 로테이션
- config.json의 `enable_log` 옵션 (기본 true)

## 수정 파일

- `mcp/context_search/server.py` — related_classes 메타데이터, 결과 수 제한, 캐시/폴백, cache_context 도구, 로깅
- `mcp/context_search/web_ui.py` — (신규) 웹 UI HTML 분리
- `watcher/watch.py` — 방화벽 설정, config 검증, git pull 안내, encoding 수정
- `watcher/common.py` — Gemini stdin 전달, LLM 교차 폴백, 날짜별 로그 시스템
- `watcher/context.py` — stem 단위 그룹핑
- `build.bat` — web_ui 모듈 포함
- `CLAUDE.md` — 에이전트 행동 원칙, 캐시/폴백, 로그 시스템 문서화
