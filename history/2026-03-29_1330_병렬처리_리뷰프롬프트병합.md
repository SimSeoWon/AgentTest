# 2026-03-29 13:30 — 병렬 처리 및 리뷰 프롬프트 병합

## 작업 개요
컨텍스트 생성과 코드 리뷰를 ThreadPoolExecutor로 병렬 실행.
코드 리뷰의 규약 검토 + 코드 검증을 하나의 프롬프트로 병합하여 LLM 호출 횟수 절반 감소.

## 변경 전후 (22개 파일 기준)

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| 컨텍스트 생성 | 순차 12회 ≈ 8분 | 병렬 5그룹 × 4워커 ≈ 40초 |
| 코드 리뷰 | 순차 36회(18×2) ≈ 15분 | 병렬 18회(프롬프트 병합) × 4워커 ≈ 2분 |
| **총 소요 시간** | **~23분** | **~3분** |
| Claude 호출 횟수 | 48회 | 23회 |

## 생성/수정 파일 목록

| 파일 | 변경 내용 |
|------|----------|
| `watcher/watch.py` | `concurrent.futures` import, `MAX_WORKERS=4`, `_process_context_group()` 분리, `_review_single_file()` 분리 (규약+검증 병합), `update_context()`/`run_code_review()` ThreadPoolExecutor 적용 |

## 주요 설계 결정

| 결정 | 이유 |
|------|------|
| MAX_WORKERS=4 | Claude API 동시 호출 부하와 속도의 균형 |
| 규약+검증 프롬프트 병합 | LLM 호출 50% 감소, Claude는 멀티태스크 프롬프트 처리 능력 충분 |

| 컨텍스트 MD 조회 시 디렉토리 단위 우선 | 새 방식 MD 우선, 기존 클래스 단위 MD 폴백 |
