# 2026-03-28 16:15 — Gemini 옵션 분석 엔진 전환 기능 추가

## 작업 개요
코드 리뷰 / 에셋 검증 분석 엔진을 Claude ↔ Gemini 간 전환할 수 있는
옵션 기능 추가. Gemini CLI 미설치 환경에서는 자동으로 Claude로 폴백.

## 수정 파일
- `watcher/watch.py`
- `watcher/agent_templates.py`

## 주요 변경 내용

### watch.py

**신규 함수**
- `_call_gemini(prompt)` — Gemini CLI 호출. 미설치 시 Claude로 자동 폴백
- `_call_llm(prompt, use_gemini)` — 플래그에 따라 Gemini/Claude 디스패치

**수정 함수**
- `load_or_init_config()` — Gemini CLI 감지 시 `use_gemini` 옵션 입력 추가
- `run_code_review()` — `use_gemini` 파라미터 추가, `_call_llm()` 사용
- `_build_review_report()` — 동일
- `run_asset_validation()` — 동일
- `main()` — config에서 `use_gemini` 추출, 시작 로그에 분석 엔진 표시

**config.json 변경**
```json
{
  "use_gemini": false,   ← 신규 (Gemini CLI 있을 때만 질문)
  ...
}
```

**폴백 우선순위**
```
use_gemini: true + gemini 설치됨 → Gemini
use_gemini: true + gemini 없음   → Claude (자동 폴백)
use_gemini: false                → Claude
```

### agent_templates.py
gemini-query MCP를 전 에이전트에 등록 (기존 02, 07 외 추가):
- 03_코드규약, 05_코드검증 — 신규 추가
- 08_로그분석, 09_크래시분석, 10_에셋검증 — 신규 추가

서브 에이전트가 gemini_analyze 툴로 Gemini에 분석을 위임하는
패턴이 가능해짐 (Claude = 오케스트레이션, Gemini = 분석 처리)
